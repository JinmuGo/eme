What eme does

Mission control

One dashboard shows every agent as idle, working, or waiting-for-input. Jump straight to the one that needs you.

Native to your tmux

Agents run in your real tmux sessions and windows — copy-mode, popups, your own keybindings all work. Not a hidden tmux a TUI owns, not an Electron app.

git worktree-native

Each agent gets an isolated worktree and its own tmux window. folder = project = session, with no registration step.

Stays out of your way

No diff/merge/approval GUI. Review stays where it belongs — in your editor and in git.

How it works

eme is native to tmux. Each project you open becomes a tmux session, and every git worktree inside it becomes a tmux window with the agent running in its pane — your real tmux, not a hidden one a TUI controls. To show status, eme reads the foreground process in each pane, which tells running from exited but cannot tell whether an agent is working or waiting for you.

Running eme hooks install closes that gap: it wires the agent's lifecycle hooks — Claude Code today — to push its true state into a tmux pane option that eme reads as ground truth. A prompt you submit marks the agent running, a permission request or a question marks it waiting, and finishing a turn marks it idle. The dashboard re-reads tmux on every refresh, tallies how many agents are waiting or crashed, and lets you jump straight into the one that needs you. Nothing about your agent changes — eme only observes and moves your tmux client.

FAQ

What is eme?

eme is a free, open-source (MIT) terminal tool that runs each AI coding agent in its own git worktree as a real tmux session, and shows across every worktree which agent is waiting for input, which is running, and which is idle.

Which AI coding agents does eme support?

eme ships with claude, codex, gemini, and opencode built in, and runs any other command-line agent on your PATH. To add or override one, edit ~/.config/eme/config.toml and add an [[agents]] entry with a name and the command eme should launch — matching a built-in name overrides it.

How does eme know an agent is waiting for me?

eme reads each tmux pane's foreground process for a base reading. Run eme hooks install and the agent pushes its exact state into a tmux pane option that eme treats as ground truth: a permission prompt or a question shows as waiting, an active turn as running, and a finished turn as idle. Hooks support Claude Code today; other agents use the foreground heuristic.

How do I install eme?

Install with Homebrew (brew install alderwork/tap/eme), the install script (curl -fsSL https://eme.jinmu.me/install.sh | sh), or Go (go install github.com/alderwork/eme/cmd/eme@latest). eme requires tmux 3.0+ and git 2.30+. Then run eme to open the dashboard.

What platforms does eme run on?

macOS and Linux. eme is tmux-native: each project maps to a tmux session and each git worktree to a tmux window, operating on your current tmux server rather than a hidden one.

Is eme free and open source?

Yes. eme is MIT-licensed and developed in the open at github.com/alderwork/eme.

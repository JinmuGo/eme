Mission control

One dashboard shows every agent as idle, working, or waiting-for-input. Jump straight to the one that needs you.

Native to your tmux

Agents run in your real tmux sessions and windows — copy-mode, popups, your own keybindings all work. Not a hidden tmux a TUI owns, not an Electron app.

git worktree-native

Each agent gets an isolated worktree and its own tmux window. folder = project = session, with no registration step.

Stays out of your way

No diff/merge/approval GUI. Review stays where it belongs — in your editor and in git.

Dashboard keys

nNew project — fuzzy folder picker, creates main
cCreate a worktree (new branch + tmux window)
aToggle the agent in the selected worktree
APick which agent to launch from the catalog
sFloat the agents that need you to the top
pLive side preview of the selected agent
⏎Switch your tmux client to that worktree
dKill a worktree (with confirmation)

Requires tmux ≥ 3.0, git ≥ 2.30, and an AI agent on your PATH (opencode, claude, codex, …).
